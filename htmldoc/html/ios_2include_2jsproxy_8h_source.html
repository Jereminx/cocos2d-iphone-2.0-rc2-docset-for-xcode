00018 /*
00019  * A proxy is a JSObject that implements generic behavior by providing custom
00020  * implementations for each object trap. The implementation for each trap is
00021  * provided by a C++ object stored on the proxy, known as its handler.
00022  *
00023  * A major use case for proxies is to forward each trap to another object,
00024  * known as its target. The target can be an arbitrary C++ object. Not every
00025  * proxy has the notion of a target, however.
00026  *
00027  * Proxy traps are grouped into fundamental and derived traps. Every proxy has
00028  * to at least provide implementations for the fundamental traps, but the
00029  * derived traps can be implemented in terms of the fundamental ones.
00030  *
00031  * To minimize code duplication, a set of abstract proxy handler classes is
00032  * provided, from which other handlers may inherit. These abstract classes
00033  * are organized in the following hierarchy:
00034  *
00035  * BaseProxyHandler
00036  * |
00037  * IndirectProxyHandler
00038  * |                    
00039  * DirectProxyHandler
00040  */
00041

00045  * default implementation for the fundamental traps. It does, however, implement
00046  * the derived traps in terms of the fundamental ones. This allows consumers of
00047  * this class to define any custom behavior they want.
00048  */
