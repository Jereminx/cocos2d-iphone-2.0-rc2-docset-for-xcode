\subsection{Detailed Description}
\subsubsection*{template$<$typename T$>$class mozilla\-::\-Out\-Param\-Ref$<$ T $>$}

\hyperlink{classmozilla_1_1_out_param_ref}{Out\-Param\-Ref} is a wrapper that tracks a refcounted pointer passed as an outparam argument to a function. \hyperlink{classmozilla_1_1_out_param_ref}{Out\-Param\-Ref} implements C\-O\-M T$\ast$$\ast$ outparam semantics\-: this requires the callee to Add\-Ref() the T$\ast$ returned through the T$\ast$$\ast$ outparam on behalf of the caller. This means the caller (through \hyperlink{classmozilla_1_1_out_param_ref}{Out\-Param\-Ref}) must Release() the old object contained in the tracked \hyperlink{classmozilla_1_1_ref_ptr}{Ref\-Ptr}. It's O\-K if the callee returns the same T$\ast$ passed to it through the T$\ast$$\ast$ outparam, as long as the callee obeys the C\-O\-M discipline.

Prefer returning Temporary\-Ref$<$\-T$>$ from functions over creating T$\ast$$\ast$ outparams and passing Out\-Param\-Ref$<$\-T$>$ to T$\ast$$\ast$. Prefer Ref\-Ptr$<$\-T$>$$\ast$ outparams over T$\ast$$\ast$ outparams.
