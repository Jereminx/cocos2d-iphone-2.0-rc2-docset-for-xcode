00026 /* -------------------------------------------------------------------
00027    The basic API is described in the README file.
00028 
00029    The raw API is designed to bypass some of the argument packing
00030    and unpacking on architectures for which it can be avoided.
00031 
00032    The closure API allows interpreted functions to be packaged up
00033    inside a C function pointer, so that they can be called as C functions,
00034    with no understanding on the client side that they are interpreted.
00035    It can also be used in other cases in which it is necessary to package
00036    up a user specified parameter and a function pointer as a single
00037    function pointer.
00038 
00039    The closure API must be implemented in order to get its functionality,
00040    e.g. for use by gij.  Routines are provided to emulate the raw API
00041    if the underlying platform doesn't allow faster implementation.
00042 
00043    More details on the raw and cloure API can be found in:
00044 
00045    http://gcc.gnu.org/ml/java/1999-q3/msg00138.html
00046 
00047    and
00048 
00049    http://gcc.gnu.org/ml/java/1999-q3/msg00174.html
00050    -------------------------------------------------------------------- */
